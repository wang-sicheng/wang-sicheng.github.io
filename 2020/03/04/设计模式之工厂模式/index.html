设计模式之工厂模式

发表于 2020-03-04 分类于 Java

工厂模式作用是帮助我们将对象实例化的部分抽离出来，从而降低代码的耦合度，增加系统的可扩展性。

一、简单工厂模式（静态工厂模式）

  1. 产品接口类，用于区分冰激凌口味

  2. 产品实现类

  3. 工厂类，根据需要制作冰激凌

  4. 消费者类

    特点：一个工厂生产多种产品
    优点：简单，封装了生产细节
    缺点：违背了开闭原则，扩展性差

二、工厂方法模式

  1. 产品接口类，产品实现类同上

  2. 工厂接口类

  3. 工厂实现类

  4. 消费者类

特点：多个工厂，每个工厂只生产一种产品
优点：符合开闭原则，扩展性更好

三、抽象工厂模式

  1. 产品接口类
  2. 产品实现类
  3. 工厂接口类
  4. 工厂实现类
  5. 消费者类 特点：存在多个抽象产品类，每个抽象产品类可以派生出多个具体产品类。一个具体的工厂生产多种产品。
